# Governance

## Current model

AI Economy Ledger is presently a maintainer-led project. The repository owner is the final decision-maker and only direct writer to protected branches. Contributions flow through issues and pull requests.

## Decision process

- Routine implementation decisions are resolved in pull-request review.
- Material architecture, methodology, licensing, security, or governance changes require a decision-log entry.
- Evidence and documented project principles take precedence over contributor seniority.
- Unresolved high-impact disagreements remain unmerged until the maintainer decides or requests additional review.

## Future roles

As participation grows, the project may add contributor, reviewer, domain steward, security responder, and maintainer roles. Access should follow demonstrated responsibility and least privilege.

## Release authority

Only the maintainer may approve production deployment, database migration, domain change, credential rotation, or release publication until a documented delegation model is adopted.

## Changes to governance

Governance changes require a focused pull request, rationale, public review period when practical, and an entry in `docs/DECISION_LOG.md`.
